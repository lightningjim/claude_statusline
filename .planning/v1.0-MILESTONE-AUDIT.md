## Cross-Phase Integration

Integration checker verdict: **PASSED** — 13/13 wires connected, 0 orphaned, 0 missing.

- `render_top_line` composes segments in the correct order: project → git → gsd → model → weather, None-filtered cleanly (D-09/D-10 verified positionally by tests).
- `render_bottom_line` composes context bar + 5h/weekly rate segments, honoring `bar_style` and `thresholds`.
- A single `load_config()` produces one `cfg` dict consumed by every segment builder. All config reads use `.get(key, default)` — no segment can `KeyError` on absent config; `load_config` returns `deepcopy(DEFAULTS)` on any file error.
- Weather pipeline wired end-to-end: cache read → fire-and-forget `maybe_spawn_refresh` → `run_refresh` (`--refresh`) under an `O_CREAT|O_EXCL` lock → `fetch_weather`/`fetch_alerts`. Render path never blocks on network.

### E2E Flows (5/5 complete)

1. **Standard render** — `echo '<json>' | python3 claude-statusline.py` → two-line bar, exit 0.
2. **Empty stdin** — `_load_stdin()` → `{}`; segments degrade to None; exit 0, no traceback.
3. **Non-repo dir** — git + gsd segments return None; bar still renders; exit 0.
4. **Toggles off** — `show_git=false, show_gsd=false` → exact Phase-1 top-line string preserved.
5. **No venv / no astral** — `_WEATHER_OK=False` → weather segment None; Phase-1 bar unaffected.

## Tech Debt (non-blocking)

1. **pyproject.toml version skew** — declares `0.1.0` while the runtime `_APP_VERSION = "0.2.0"` drives the NWS User-Agent. Runtime uses the script constant, so no functional break; metadata is inconsistent.
2. **Installed venv may lack `astral`** — the integration checker reported `~/.claude/claude-statusline/.venv/` raises `ModuleNotFoundError` for `astral`. If accurate, the live weather/sun layer (WX-01..06) would silently degrade to nothing in the actual install. Weather rendered correctly and was human-approved at the Phase 2 checkpoint (2026-05-28), so the venv was complete then — **recommend re-verifying the current install** (`~/.claude/claude-statusline/.venv/bin/python -c "import astral, requests"`).
3. **Empty SUMMARY `requirements-completed` frontmatter** — populated for only 5 of 19 requirements. VERIFICATION.md and REQUIREMENTS.md both confirm full coverage; tracking-field gap only.
4. **Weather path not auto-tested in system Python** — 53 astral-gated tests self-skip on every system-Python pytest run; the weather layer is exercised only in the in-venv run (256 passed, Phase 2) and via the human-verify checkpoint.
5. **Stale REQUIREMENTS.md footer** — says "Last updated 2026-05-28 after Plan 01-01" though all checkboxes are `[x]` and the table reads Done/Complete. Content correct; footer note stale.

These are advisory. None block milestone completion.
