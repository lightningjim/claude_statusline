<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| network response → OSC 8 URL field | `id` (status.claude.com Statuspage summary) and `UGC` (api.weather.gov CAP) are network-controlled values destined for the OSC 8 URI field, which is interpreted by the terminal emulator |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01 | Tampering | `_valid_incident_id` / `_valid_ugc` (URL components) | mitigate | Allowlist (`re.fullmatch`) the id (`^[0-9a-z]+$`) and UGC (`^[A-Z]{2}[CZ][0-9]{3}$`) BEFORE embedding; any ESC/ST/BEL/control byte fails the match → None → plain text. Allowlist, not the existing denylist strip. (Task 3) |
| T-09-02 | Tampering | `osc8()` emission | mitigate | osc8 returns text unchanged on falsy enabled/url, so a rejected component (None url) yields zero `\x1b]8` bytes — the breakout path is closed at the single emission point. (Task 1) |
| T-09-SC | Tampering | npm/pip/cargo installs | accept | No new packages installed this phase — stdlib only (`re`, `os` already imported). No package-legitimacy gate required. |
</threat_model>
