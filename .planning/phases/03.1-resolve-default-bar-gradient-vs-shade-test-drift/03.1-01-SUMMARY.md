# Phase 03.1 Plan 01: Resolve Default Bar Gradient vs Shade Test Drift Summary

**One-liner:** Non-destructive HOME-override isolation added to two failing default-bar tests, fixing config-leak test-isolation defect without touching production code or the user's live gradient config.

## What Was Built

Resolved a test-isolation defect (D-01) where two tests in `tests/test_bottom_line.py` were
reading the developer's live `bar_style = "gradient"` config and failing their `▓` assertions
against the shipped `"shade"` default. The fix adds a non-destructive `_NO_CONFIG_HOME` helper
(empty tempdir, no TOML) that forces `load_config`'s silent DEFAULTS fallback, then routes the
two named tests and `TestBottomLineSynthetic._run` through it. Production code (`claude-statusline.py`)
is byte-for-byte unchanged.
