## Summary

The phase adds a module-level `_NO_CONFIG_HOME` tempdir and an optional `home:
str | None = None` parameter to `run_script`, then routes three test sites
through `home=_NO_CONFIG_HOME` so `load_config` silently falls back to
`DEFAULTS["bar_style"]="shade"` rather than reading the developer's live
`~/.claude/claude-statusline/claude-statusline.toml`.  Two anti-gradient-glyph
assertions are also added to make the failure mode self-documenting.

All 48 tests in `tests/test_bottom_line.py` pass after the change.  The
isolation mechanism is sound and the new assertions prove the shade default
non-tautologically.  No blockers or warnings were found.

### Specific checks from scope

1. **HOME override isolation** — `run_script` does `env = dict(os.environ)`
   then sets `env["HOME"]` before passing it to `subprocess.run(env=env)`.
   This preserves `PATH`, `PYTHONPATH`, and all other env vars.
   `load_config` calls `os.path.expanduser("~/.claude/...")` at call time
   (not module-import time), so the subprocess resolves the config path
   against the overridden HOME correctly.  Module-level constants
   `_CACHE_PATH` (line 250) and `_LOCK_PATH` (line 346) are evaluated at
   subprocess-module-import time; since HOME is already set before the process
   starts, those paths also land inside `_NO_CONFIG_HOME`, where the
   missing subdirectory causes silent fallbacks per the existing RUN-02 policy.
   Isolation is correct.

2. **Tempdir lifecycle** — `_NO_CONFIG_HOME` is never explicitly cleaned up;
   the same pattern is used by `_ISOLATED_HOME` and `_NO_GIT_HOME` in
   `tests/test_skeleton_render.py`, so this is established convention, not
   a new defect.  The OS reclaims the empty directory on reboot.  For a
   personal project this is acceptable.

3. **Assertion correctness** — The two new glyph-exclusion loops check each
   glyph in `"█▏▎▍▌▋▊▉"`.  This set is the exact union of
   `_GRADIENT_PARTIAL` (seven chars, lines 108 of production script) plus the
   full-block `"█"` used by the gradient filled track.  The shade default
   uses only `"▓"` (filled) and `"░"` (empty), neither of which appears in
   the exclusion set.  The assertions are non-tautological: they would fail if
   the gradient config leaked through, and they would not spuriously fail on
   a correct shade render.

4. **Backward compat** — The default `home=None` leaves existing callers
   unchanged.  The eleven callers that do not pass `home=` (bracket/pct/glyph
   tests, `test_empty_stdin_still_exits_zero`, etc.) continue using the real
   `$HOME`, which is correct because those tests do not assert on bar glyph
   identity.

5. **Pre-existing lint notes (not introduced by this diff):**

   - `timezone` is imported but never used (`from datetime import datetime,
     timezone`, line 16).  Pre-existing — present in `d1e3969`.
   - `stderr` is unpacked from `_run_with_toml(...)` at lines 503, 516, 529
     but never read within those test methods.  Pre-existing — present in
     `d1e3969`.

   Both items are flagged below as INFO for completeness.
