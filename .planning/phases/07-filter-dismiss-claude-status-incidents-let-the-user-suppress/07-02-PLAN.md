<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user-supplied ignore_title_patterns → regex engine | A malicious/incompetent pattern is matched against incident titles on the derivation path. |
| status feed titles → pattern matcher | Untrusted titles are the match target; an adversarial title must not cause pathological matching. |
| dismissal store → suppression decision | A corrupt/tampered store must not hide a real (especially escalating) incident. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-04 | Denial of Service | _is_suppressed regex match (ReDoS / catastrophic backtracking) | mitigate | Default matching is case-insensitive SUBSTRING (no backtracking). When a pattern is treated as regex, re.compile + re.search run inside a per-pattern try/except that catches re.error and falls back to substring; any failure degrades to no-match. A bad/evil pattern never hangs or crashes the derivation (runtime contract: render must never hang). |
| T-07-05 | Information Disclosure | escalation bypass via stale dismissal | mitigate | Escalation re-surface (D-03): live_rank > stored_rank → dismissal void → incident shown. A muted minor that goes major/critical is never hidden. Tested explicitly. |
| T-07-06 | Tampering | corrupt/non-dict dismissals fed to derivation | mitigate | _is_suppressed normalizes non-dict dismissals and wraps its body in try/except → False (not suppressed). Corrupt store degrades to no suppression — fails safe (shows incidents) rather than hiding them. |
| T-07-07 | Tampering | auto-prune writing the wrong file | mitigate | Prune writes ONLY the tool-owned _DISMISSALS_PATH via write_dismissals (atomic os.replace); the user's TOML is never written (D-05). Prune runs only in the detached --refresh child, never on the render path. |
| T-07-SC | Tampering | npm/pip/cargo installs | mitigate | No new dependencies — `re` is stdlib (already available). No package installs; no legitimacy gate required. |
</threat_model>
