<objective>
Close the single UAT gap on Phase 07: `--dismiss <id>` (and editing `ignore_title_patterns`)
does not suppress an incident on the bar immediately — suppression is currently computed only at
FETCH time and baked into the cache (`noteworthy`/`label`), while the render path
`_claude_status_segment` reads the baked value and never re-applies the filter. A purely-local,
zero-network action therefore cannot take effect until a fresh NETWORK fetch runs (and manual
`--refresh` is unreliable under the stampede lock).

DECIDED FIX (user, 2026-06-17, do NOT re-litigate): RENDER-TIME SUPPRESSION. Re-apply the
existing `_is_suppressed` logic at render time inside `_claude_status_segment` over the cached
`tracked_incidents` list plus the live dismissal store and `ignore_title_patterns` config, so
local actions take effect on the very next render — instant, zero-network, lock-independent.
Also correct the misleading `--refresh` help/confirmation text.

Purpose: Make "lit = something I should actually care about" true for local mutes, restoring the
core value of the Phase 6/7 segment.
Output: Render-time suppression layered on top of the existing fetch-time baking; corrected help
text; unit tests proving instant suppression, escalation re-surface, and never-crash.

NON-NEGOTIABLE constraints from the fix decision:
- REUSE `_is_suppressed` — the single suppression ordering. Do NOT fork or duplicate the
  suppression/escalation logic (see the comment at claude-statusline.py:1556-1557: only ONE
  ordering must exist, shared by `_is_suppressed` and `_derive_claude_status`).
- Preserve the D-03 escalation safety valve: `_is_suppressed` already compares live impact to
  `impact_at_dismiss` via `_CLAUDE_IMPACT_RANK`; render-time suppression inherits this by calling
  `_is_suppressed` with the live impact from each cached tracked incident.
- Preserve Phase 6 quiet-when-healthy (noteworthy=False / cold / stale -> None) and never-crash
  (body in try/except -> None). The render path must perform NO network I/O and stay fast — only
  cheap local file reads (cache + dismissal store).
- Do NOT change the stampede-lock behavior of `run_refresh` (explicitly out of scope per the
  decision). Fetch-time baking remains as-is; render-time suppression layers on top.
</objective>
