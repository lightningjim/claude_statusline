<objective>
Add the three management CLI flags that mirror the existing `--refresh` pattern:
`--status-incidents` (the visibility surface — list tracked incidents and their
dismissed/stale state so the user learns ids and picks keywords), `--dismiss <id>`,
and `--undismiss <id>` (mutate the tool-owned dismissal store and exit). None read
stdin or print the status bar.

Purpose: This is how the user actually drives the filter built in Plans 01-02 —
discovering ids, muting the perpetual incident, and lifting a mute. It is the only
human-facing surface of the phase.

Output: Three `main()` arg branches (hand-rolled sys.argv, no argparse), an
ANSI-sanitizing incident-table printer reading cache + store only, and tests for
store mutation, table output, and graceful missing-arg handling.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| sys.argv → store mutation | A user-supplied id string is written into the tool-owned store. |
| cached incident titles → terminal (--status-incidents) | Untrusted upstream titles are printed to the user's terminal. |
| missing/extra CLI args → main() | Malformed invocation must not crash the tool. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-08 | Injection | --status-incidents title printing | mitigate | Titles ANSI-stripped + width-bounded VERBATIM from _claude_status_segment before stdout — no raw `\x1b` reaches the terminal (preserves Phase 6 T-06-04). Tested with status_malicious_title.json. |
| T-07-09 | Denial of Service | malformed CLI invocation (missing id) | mitigate | idx+1 bounds-guarded; missing value → usage hint + clean exit, never IndexError/traceback. The status bar (other invocations) is unaffected since these branches exit early. |
| T-07-10 | Tampering | --dismiss writing an arbitrary id | accept | An arbitrary id only mutes a non-existent incident in the tool-owned store; auto-prune (Plan 02) removes ids absent from the feed. No impact on the user's TOML or on real incidents. Low risk, local single-user tool. |
| T-07-SC | Tampering | npm/pip/cargo installs | mitigate | No new dependencies — stdlib only (sys/json/os). No package installs; no legitimacy gate required. |
