<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| status.claude.com feed → cache payload | Untrusted incident titles/ids flow into the widened cache list (printed later by Plan 03). |
| dismissal store file → process | Tool-owned JSON on disk; may be hand-tampered, corrupt, or oversized. |
| concurrent --refresh / render writes | Two processes may write the store/cache near-simultaneously. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Tampering | status_dismissals.json read | mitigate | read_dismissals wraps json.load in try/except and rejects non-dict → returns {} (no suppression). Corrupt/oversized/hand-edited file degrades, never raises. |
| T-07-02 | Denial of Service | concurrent store/cache writes | mitigate | write_dismissals copies the temp-file + os.replace atomic pattern from write_cache_section (:301-306); render never reads a half-written file. |
| T-07-03 | Injection | RAW titles widened into cache payload | accept (this plan) / mitigate (Plan 03) | Titles stored RAW here (matches Phase 6 cache contract, T-06-01). Sanitization is enforced where titles reach the terminal — Plan 03 --status-incidents must ANSI-sanitize before print. Flagged forward. |
| T-07-SC | Tampering | npm/pip/cargo installs | mitigate | No new dependencies (stdlib json/os/time only). No package installs in this plan; no legitimacy gate required. |
</threat_model>
