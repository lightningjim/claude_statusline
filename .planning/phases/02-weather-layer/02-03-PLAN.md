<objective>
Add the active-alert override: when NWS reports an active alert for the configured location, the trailing sun-event detail in the weather segment is replaced by a warning glyph plus the full NWS event name, severity-colored, with a +N suffix when multiple alerts are active. Alerts are fetched in the same lock-guarded detached background child (cached under the alerts section) and deduped via the CAP references-chain algorithm so superseded/cancelled/expired alerts never show. The render reads the cached alerts and never blocks; when no qualifying alert exists the trailing detail falls back to the sun event.

Purpose: Delivers WX-04 (alert override). Builds on Plan 02's fetch/cache/segment infrastructure. Implements D2-11 (severity coloring, highest-severity + N, dedup reuse) and the alert portion of D2-12 (stale-OK up to alerts_max_stale, else sun fallback). The user is a meteorologist — preserve real NWS event names and severity fidelity; do not dumb down.

Output: alerts fetch + dedup + severity selection + override rendering, fixture-driven offline tests.
</objective>

<alerts_reference>
<!-- Alerts endpoint + dedup algorithm + severity fields. The WxDesktopPy dedup is over an Alert dataclass; extract the ALGORITHM, simplify to operate on plain dicts (no dataclass/pint/logging). -->

Endpoint: GET https://api.weather.gov/alerts/active?point={lat:.4f},{lon:.4f} with header Accept: application/ld+json. Empty result (no alerts) is a valid 200, not an error.

Per-alert fields (NWS CAP, under each feature's properties): event (full name string), severity ("Extreme"/"Severe"/"Moderate"/"Minor"/"Unknown"), messageType (Alert/Update/Cancel/Ack/Error), references (list of prior alert identifiers this one supersedes), expires (ISO8601), sent (ISO8601), id/identifier.

Dedup algorithm (port from WxDesktopPy dedup.py, operate on dicts):
  1. Build a superseded set: every identifier appearing in any alert's references, PLUS every alert whose messageType is in {Cancel, Ack, Error}.
  2. Drop expired alerts (expires < now).
  3. Sort the remainder by sent descending.

Severity rank for D2-11 selection: Extreme > Severe > Moderate > Minor > Unknown. Show the highest-severity surviving alert as the warning glyph + event name; append " +N" where N is the count of other surviving alerts. Severity color: Extreme/Severe -> RED, Moderate/Minor -> YELLOW (Unknown -> YELLOW or DIM, Claude's discretion).
