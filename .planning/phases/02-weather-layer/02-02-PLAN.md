- geo (cache near-permanently): GET https://api.weather.gov/points/{lat:.4f},{lon:.4f} -> properties.cwa (a.k.a. gridId), properties.gridX, properties.gridY, properties.observationStations (a URL)

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| api.weather.gov -> background fetch | Untrusted external JSON parsed by the detached child; must never crash and must bound size/time |
| render -> detached subprocess | Render spawns a child under the venv; the argument vector must be fixed (no shell interpolation) |
| filesystem -> cache.json / lockfile | Fixed paths under the install subfolder; atomic writes; no untrusted path components |
| config -> User-Agent header | contact_email flows from config into the NWS request header; must not leak to stdout/logs |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-06 | Information Disclosure | contact_email in NWS User-Agent | mitigate | Email is read from config and placed only in the User-Agent request header; never printed to stdout; render-path code does not log it |
| T-02-07 | Denial of Service | parsing untrusted NWS JSON | mitigate | All fetch/parse wrapped in try/except returning None (cache unchanged); requests timeout bounds fetch time; only runs in the detached child, never the render path |
| T-02-08 | Tampering | detached refresh subprocess | mitigate | subprocess.Popen with a FIXED argv [sys.executable, __file__, "--refresh"], no shell=True, no interpolated input; runs under the venv interpreter |
| T-02-09 | Denial of Service | concurrent refresh stampede | mitigate | Lockfile created with O_CREAT|O_EXCL (or fcntl.flock); a second concurrent fetch exits immediately when the lock is held |
| T-02-10 | Tampering | half-written cache.json | mitigate | write_cache_section writes a temp file then os.replace (atomic); a render never reads a partial file |
| T-02-11 | Spoofing | NWS rejects requests without User-Agent (403) | mitigate | Every NWS request sends the NWS-ToS-compliant User-Agent (app + contact email); fetch fails closed (cache unchanged) on any non-200 |
</threat_model>
