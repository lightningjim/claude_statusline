# Phase 07.1 Plan 01: Resolved-incident derivation + cache contract Summary

**One-liner:** Resolved-incident kind/severity flowing through `_derive_claude_status` Rule 3, GREEN hue in `_claude_status_color`, and resolved entries widened into `_collect_tracked_incidents` — cache contract stable for Wave-3 render.

## What Was Built

This plan established the Wave-1 resolved-incident DERIVATION + CACHE contract:

**Task 1 — GREEN resolved hue in `_claude_status_color` (D-03):**
Added `"resolved": GREEN` to the `hue_map` dict (reuses existing `GREEN` constant at line 77; no new constant). Existing `minor`/`major`/`critical`/`maintenance` mappings and the `YELLOW` safe default are unchanged. Intent comment matches the adjacent `maintenance` comment style.

**Task 2 — Resolved-vs-red branch in `_derive_claude_status` Rule 3 (D-04/D-05/D-06):**
Inside Rule 3, before returning the red degraded result, a resolved-incident scan now runs for the specific degraded `comp_name`. The scan copies the Rule 1 matching shape but inverts the status filter to `inc_status != "resolved"`. It intersects `_tracked_component_names(...)` with the specific degraded component (D-05: only this component counts), gates on the same `_is_suppressed(...)` call Rule 1 uses (D-06: single suppression ordering), and picks the newest by `updated_at` string. If a match is found: returns `{"severity":"resolved","label":<raw title>,"kind":"resolved"}`. No match: falls through to the existing red `{"severity":...,"kind":"degraded"}` return. The resolved scan is wrapped in its own `try/except` — a failure degrades to the existing red return, never raises (D-10). Rule 1 (active incident) and Rule 2 (maintenance) ordering preserved.

**Task 3 — Widen `_collect_tracked_incidents` to carry resolved incidents (D-06/D-07):**
Added `"resolved"` to the status filter alongside the three unresolved statuses. The per-item dict shape is unchanged: `{id, impact, status, title, component}` — `status` now may carry `"resolved"`, which Wave-3 render and CLI branch on. Per-item `try/except`, outer `return []`, and the RAW title contract are all preserved.

**New fixture `tests/fixtures/status_resolved_degraded.json`:** A resolved incident (`inc-resolved-001`, status=`resolved`, impact=`major`, updated_at=`2026-06-18T12:00:00Z`) touching Claude Code which is in `partial_outage`. Mirrors the `status_incident_tracked.json` shape exactly.
