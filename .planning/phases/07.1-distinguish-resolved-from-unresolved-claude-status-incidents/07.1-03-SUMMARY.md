# Phase 07.1 Plan 03: Resolved render branch + CLI STATE Summary

**One-liner:** Resolved baked items render GREEN + check-circle + `resolved:` prefix in `_claude_status_segment` (D-03); D-06 Risk #2 fall-through guard ensures muted degraded/resolved items return None; `--status-incidents` STATE column shows `resolved` with `dismissed` precedence (D-07).

## What Was Built

This plan completed the Wave-3 render + CLI edge of the resolved-incident pipeline:

**Task 1 — Resolved render branch + D-06 Risk #2 fix in `_claude_status_segment`:**

Two coordinated edits to `_claude_status_segment`:

**Change A — Step 3b baked-kind guard (Risk #2 fix):** The surviving-incident block previously hardcoded `_kind_override = "incident"` for all baked item types. This caused a resolved baked item with a non-suppressed tracked incident to render as a red incident (overriding the baked `kind="resolved"` signal). The fix adds a `baked_kind` check: only when `baked_kind == "incident"` are the severity/label/kind overridden from the surviving incident; for `degraded` and `resolved` baked items, the baked values pass through unchanged. The mute-gate (`surviving_inc is None → return None`) is preserved verbatim for all baked kinds — so a muted degraded/resolved item still returns None (D-06 Risk #2 closed).

**Change B — Step 5: Resolved glyph/color branch:** Added a new `elif kind == "resolved" or severity == "resolved"` branch immediately before the incident/degraded `else`, mirroring the WR-01 dual-key guard used for maintenance (line 3739). Nerd glyph = `_NF_GSD_DONE` (reused directly — check-circle, already in repo); emoji fallback = `"\U00002705"` (✅, literal-with-comment style matching existing fallbacks); `color = _claude_status_color(severity)` → returns GREEN (added in Wave 1). **Change C — Step 7: `resolved:` prefix:** For resolved kind/severity, assembles `detail = f"{glyph} resolved: {safe_label}"` instead of `f"{glyph} {safe_label}"`. The prefix is applied here (render concern) — not baked into the cached label (RAW-in-cache contract preserved). The Step 6 sanitizer and empty-label fallback are unchanged.

**Task 2 — `resolved` STATE in `_print_status_incidents` (D-07):**

Extended the per-entry STATE computation (previously `"dismissed" if raw_id in dismissals else "active"`) to a three-way:
```python
if raw_id in dismissals:
    state = "dismissed"
elif entry.get("status") == "resolved":
    state = "resolved"
else:
    state = "active"
```
`dismissed` retains highest precedence. `COL_STATE = 10` already fits `"resolved"`. Updated the docstring STATE-values list to document `"resolved"` and its precedence relative to `"dismissed"`. All other code paths (stale loop, `_sanitize`, header/separator, outer `try/except`) untouched.
