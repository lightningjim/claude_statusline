<decisions>
## Implementation Decisions

### Segment scope
- **D-01:** Four segments convert to Nerd Font glyphs: **weather conditions/alerts**, **sun
  events**, the **thinking indicator**, and the **rate-limit glyphs**. The sun events use the
  purpose-built `wi-sunrise` / `wi-sunset` glyphs, keeping them visually unified with the
  weather block they sit in.
- **D-02:** The `▓░` context-progress bar is **explicitly NOT converted** in this phase —
  Unicode block elements render the same in any font, and block-fill presets are Phase 3.
  Do not touch `_FILLED`/`_EMPTY` here.

### Weather condition granularity
- **D-03:** **Exploit the full `wi-*` granularity** rather than a 1:1 emoji swap. Expand the
  current ~7 condition buckets to map the full NWS icon-token vocabulary — distinct glyphs for
  day vs night, and for snow / sleet / freezing-rain / rain-snow-mix / thunderstorm /
  thunderstorm-with-rain / fog-haze / windy — driven by the NWS `icon`-URL token
  (`skc/few/sct/bkn/ovc/ra/rasn/sn/fzra/tsra/fg/wind` + `day|night`). This is the
  meteorologist-fidelity payoff of the phase.
- **D-04:** **Clear/mostly-clear nights show the actual current moon phase**, not a generic
  moon. Compute the phase from `astral` (already a dependency — `astral.moon.phase()` returns
  0–27.99) and map it across the 28-glyph `wi-moon-*` set (new → waxing-crescent ×6 →
  first-quarter → waxing-gibbous ×6 → full → waning-gibbous ×6 → third-quarter →
  waning-crescent ×6). No extra network — astral is local.
- **D-05:** Preserve Phase 2's existing day/night detection (`is_night` from `/night/` in the
  NWS icon-URL path) and the text-then-URL token matching in `_icon_to_emoji` — this phase
  swaps the glyph *targets* and broadens the token table, it does not rebuild the lookup
  mechanism.

### Config & fallback
- **D-06:** Implement ENH-04's `icon_set` toggle in the TOML, but **default it to `nerd`**
  (fonts are installed). `icon_set = "emoji"` reselects the Phase 2 emoji tables, which are
  **retained in code as the fallback path** (not deleted). Lives in `DEFAULTS` alongside the
  other Phase 1/2 config.
- **D-07:** `icon_set` is a **single global switch** — one key flips all four converted
  segments (weather, sun, thinking, rate-limit) together. No per-segment toggles; never a
  half-nerd/half-emoji bar.

### Glyph coloring
- **D-08:** Apply **semantic weather colors** to the monochrome glyphs (recovering the color
  that emoji carried inherently, now with intent): sun = yellow, clear-moon = dim/white,
  rain = blue, snow = cyan, storm = magenta/red, fog = gray. **Alerts keep their existing
  Phase 2 severity coloring** (D2-11). Coloring is a top-line concern and must not be confused
  with the bottom line's reserved green/yellow/red usage-threshold semantics.

### Claude's Discretion
- Exact `wi-*` PUA codepoints for each condition/sun/moon glyph (research against the Weather
  Icons cheat sheet during planning).
- The specific Nerd Font glyph for the thinking indicator (e.g. `nf-md-brain` 󰧑, a lightbulb,
  or a thought-bubble) and for the rate-limit segments (e.g. `nf-fa-hourglass` for 5h,
  `nf-fa-calendar` for weekly).
- The precise ANSI codes / 256-color values for each semantic weather color, reusing the
  existing GREEN/YELLOW/RED/DIM constants where they fit and adding new ones (blue/cyan/
  magenta/gray) as needed.
- The internal shape of the dual glyph tables (e.g. one `nerd` table + one `emoji` table
  selected by `icon_set`, vs a per-condition record holding both glyphs).

</decisions>

<specifics>
## Specific Ideas

- The author is a **degreed meteorologist** (carried from Phase 2) — the granularity decision
  (D-03/D-04) exists to honor a forecaster's read of the sky: a clear night should show the
  *real* moon phase, freezing rain should not collapse into "rain," etc. Don't dumb the
  weather signal back down to coarse buckets.
- Target feel — the weather block keeps the Phase 2 layout
  `[<icon> <temp> | 🌧️<pop>% | <sun-or-alert>]` but every glyph in it (condition, precip,
  sun/moon, alert) becomes a crisp single-cell Nerd Font glyph with semantic color.

</specifics>

<code_context>
## Existing Code Insights

### Reusable Assets
- **`_icon_to_emoji` lookup machinery** — the lowercase contains-match over
  `text_description` then `icon_url` path, plus the `is_night` flag, is exactly the dispatch
  the broader `wi-*` table needs. Broaden the table and the day/night/moon branching; keep the
  function shape.
- **Segment-builder pattern** — each `_*_segment()` returns a string or `None`; nothing about
  the join logic changes. Glyph swaps are localized inside each builder.
- **Color constants** (GREEN/YELLOW/RED/DIM/RESET) and the never-crash/silent-omit discipline
  — reuse for semantic weather coloring; add blue/cyan/magenta/gray constants.
- **`astral`** — already imported for sun times; `astral.moon.phase()` gives the moon phase
  locally with no added dependency or network.

### Established Patterns
- Runtime contract (RUN-01/02): read stdin → write line(s) → exit fast; never emit a traceback
  that breaks the bar. A missing/unknown glyph must degrade (fallback glyph or omit), never
  crash.
- Per-segment silent omit (D-10) and the layered degradation (D2-12) remain in force.
- Single TOML config holds all toggles; `icon_set` joins the existing `[location]`/`[cache]`/
  `[units]` tables.

### Integration Points
- Output consumed by Claude Code's `statusLine`; stdout only. No terminal/font info is
  available on stdin — which is *why* `icon_set` is a manual config toggle (D-06) rather than
  runtime font detection.
- No network or cache changes: this phase only changes how already-fetched data is rendered.

</code_context>
