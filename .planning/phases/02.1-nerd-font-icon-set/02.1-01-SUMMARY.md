## What Was Built

**Task 1 — Config toggle + semantic color constants + astral moon import:**

- Added `"display": {"icon_set": "nerd"}` to `DEFAULTS` as a new top-level sub-table (mirrors `"units"` / `"weather"` shape). `_deep_merge` carries it through unchanged — no changes to config loading logic.
- Added four ANSI constants after `RESET`: `BLUE = "\033[34m"`, `CYAN = "\033[36m"`, `MAGENTA = "\033[35m"`, `GRAY = "\033[90m"` — marked top-line only (D-08).
- Added `_wx_color(condition_type)` mirroring `_alert_color` shape: storm -> MAGENTA, rain -> BLUE, snow -> CYAN, fog -> GRAY, sun -> YELLOW, unknown -> RESET; wrapped in try/except so any input degrades to RESET.
- Extended the existing astral `try` block with `from astral.moon import phase as _moon_phase` — no new dependency, `_ASTRAL_OK` guard unchanged.
- Added `# Phase 3: DO NOT modify here` comment on `_FILLED`/`_EMPTY` to reinforce D-02.

**Task 2 — Nerd Font glyph constants + 28-slot moon-phase table:**

- Added 21 named `_WI_*` / `_NF_*` glyph constants covering all D-03 condition categories with day/night splits, sun events, thinking indicator, rate-limit glyphs, and fallback.
- Added `_MOON_PHASE_GLYPHS`: 28-slot `list[str]` with wi-moon-new (U+E380) through wi-moon-waning-crescent-6 (U+E39B), following the phase sequence: new → waxing-crescent ×6 → first-quarter → waxing-gibbous ×6 → full → waning-gibbous ×6 → third-quarter → waning-crescent ×6 (D-04).
- Added `_moon_phase_index(phase: float) -> int`: `int(phase)` clamped to `[0, 27]` with `max(0, min(27, ...))`, wrapped in try/except that falls back to 0 for any invalid input (T-02.1-03).
