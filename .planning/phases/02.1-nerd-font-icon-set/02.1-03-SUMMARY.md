### Task 2 (post-checkpoint codepoint fix — b435b63)

The human-verify checkpoint identified several wrong codepoints. A fontTools cmap audit against `JetBrainsMonoNerdFontMono-Regular.ttf` confirmed the following corrections:

| Constant | Old codepoint | Old glyph name | New codepoint | Correct glyph name |
|----------|---------------|----------------|---------------|-------------------|
| `_MOON_PHASE_GLYPHS[0..27]` | U+E380..E39B | direction_down_right + clock faces | U+E38D..E3A8 | weather-moon_new through weather-moon_waning_crescent_6 |
| `_WI_FREEZING_RAIN` | U+E311 | weather-cloudy_windy | U+E316 | weather-rain_mix |
| `_WI_RAIN_SNOW` | U+E311 | weather-cloudy_windy | U+E316 | weather-rain_mix (same — intentional) |
| `_WI_THUNDERSTORM_RAIN` | U+E31E | weather-windy | U+E31C | weather-storm_showers |
| `_WI_SLEET` | U+E3AC | weather-night_alt_sleet (night variant) | U+E3AD | weather-sleet (plain) |
| `_WI_WINDY` | U+E3BC | weather-day_light_wind (day variant) | U+E31E | weather-windy (plain) |

The moon table bug was the most significant: all 28 slots were shifted 13 codepoints low, so every moon phase rendered as a directional arrow or clock-face glyph.

### Task 3 (regression guard — 6da5c42)

Added two test classes to `tests/test_nerd_icons.py`:

**`TestAllNerdGlyphConstantsInInstalledFont`**: Loads the installed Nerd Font TTF via fontTools, builds a cmap, and asserts that every `_WI_*`/`_NF_*` single-glyph constant's codepoint is present in the cmap. Guards against future wrong-codepoint introductions.

**`TestMoonPhaseGlyphsAreMoonGlyphs`**: Asserts all 28 moon table entries exist in the cmap AND that each glyph name contains `"moon"`. Dedicated sub-tests verify slot 0 contains `"moon_new"` and slot 14 contains `"moon_full"`. This is the exact check that would have caught the clock-face bug before it reached human-verify.

Both classes use `fc-list` + glob fallback to locate the font file and skip cleanly when fontTools is not installed or no font is found — they never break CI.
