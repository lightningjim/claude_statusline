## What Was Built

**Task 1 — Dual condition tables + _icon_to_glyph resolver:**

- Renamed `_NWS_ICON_MAP` → `_NWS_ICON_MAP_EMOJI` (Phase 2 emoji table retained exactly as-is for `icon_set="emoji"` fallback). The table and all its glyphs are unchanged — D-06.
- Added `_NWS_ICON_MAP_NERD`: full D-03 vocabulary using 3-element tuples `(keywords_tuple, _WI_* glyph, category_string)`. Ordering discipline preserved (specific entries before broad): thunderstorm → fzra → rasn → sleet → snow → rain → fog → wind → partly/scattered → cloudy/broken → clear/sunny.
- Added `_NERD_SUN_GLYPHS = frozenset({_WI_DAY_CLEAR, _WI_DAY_PARTLY})` for O(1) clear-night detection.
- Added `_icon_to_glyph(text_description, icon_url, icon_set="nerd")`: text-then-URL lowercase contains-match (D-05 preserved); `is_night = "/night/" in icon_path` flag preserved; for nerd path with clear/sun glyph at night — branches to live moon phase via `_moon_phase()` + `_moon_phase_index()` under `_ASTRAL_OK` guard; falls back to `_WI_NIGHT_CLEAR` if `_ASTRAL_OK` is False; wraps in try/except (never raises).
- Added `_icon_to_emoji(text, url)` alias delegating explicitly with `icon_set="emoji"` — never uses the "nerd" default so TestIconMapping in test_weather_fetch.py stays green (D-06).
- Added `_condition_category(text, url)` pure helper returning the matched category string for `_wx_color()` dispatch — kept separate from `_icon_to_glyph` so the resolver stays pure/testable.
- All tables and functions placed AFTER the `_WI_*` constants and `_MOON_PHASE_GLYPHS` they reference (forward-reference fix).

**Task 2 — Cache token migration + render-time wiring:**

- `fetch_weather`: replaced `_icon_to_emoji(text_desc, icon_url)` call and `"icon": icon_emoji` payload with raw tokens `"text_desc": text_desc, "icon_url": icon_url_raw`. Removed the baked-glyph "icon" key entirely. Updated cache schema comment.
- `_weather_segment`: reads `text_desc` + `icon_url` from cached weather section; resolves glyph at render via `_icon_to_glyph(text_desc, icon_url, icon_set)`; when `icon_set == "nerd"` wraps the glyph in `_wx_color(category) + RESET` (D-08, top-line only); when `icon_set == "emoji"` passes the glyph through without ANSI wrapping; `icon_set` read from `cfg.display.icon_set` at render time (D-07 toggle takes effect immediately).
- Backward-compat path: if cache has the old `"icon"` key (pre-migration), `_weather_segment` falls back to it gracefully.
- Alert override block left completely untouched — `_alert_color` severity coloring preserved (D2-11).
- Updated `test_weather_fetch.py::TestFetchWeather::test_fetch_weather_icon_mapped_to_emoji` → renamed `test_fetch_weather_stores_raw_nws_tokens` asserting `text_desc` + `icon_url` presence.

## Deviations from Plan

**1. [Rule 1 - Bug] Tables placed before _WI_* constants (forward-reference fix)**
- **Found during:** Task 1 GREEN
- **Issue:** The condition tables were initially placed at the same position as the original `_NWS_ICON_MAP` (~line 303), but they reference `_WI_THUNDERSTORM`, `_WI_FREEZING_RAIN`, etc. which are defined at ~line 524. Python raised `NameError: name '_WI_THUNDERSTORM' is not defined`.
- **Fix:** Moved `_NWS_ICON_MAP_EMOJI`, `_NWS_ICON_MAP_NERD`, `_NERD_SUN_GLYPHS`, `_icon_to_glyph`, `_icon_to_emoji`, `_condition_category` to after `_moon_phase_index` (after all `_WI_*` and `_MOON_PHASE_GLYPHS` they reference).
- **Files modified:** claude-statusline.py

**2. [Rule 1 - Test Correction] fzra/rasn share glyph per Weather Icons spec**
- **Found during:** Task 1 GREEN
- **Issue:** Test `test_all_granular_precip_distinct` required all 7 precip types to have distinct glyphs, but Plan 01 assigned `_WI_FREEZING_RAIN` and `_WI_RAIN_SNOW` both to `wi-rain-mix` (U+E311) — Weather Icons has no distinct codepoint for freezing rain vs rain-snow mix.
- **Fix:** Relaxed the test to require only the 5 core types (snow/sleet/thunderstorm/fog/wind) to be distinct; fzra and rasn are verified as non-empty but may share a glyph.
- **Files modified:** tests/test_nerd_icons.py

**3. [Rule 1 - Test Correction] Nerd table 3-element tuple shape**
- **Found during:** Task 1 GREEN
- **Issue:** Test `test_nerd_table_is_list_of_tuples` expected 2-element tuples (matching the emoji table), but the nerd table legitimately uses 3-element tuples to carry the category tag needed for `_wx_color`.
- **Fix:** Updated assertion to accept 2 or 3 elements with a clear comment.
- **Files modified:** tests/test_nerd_icons.py
