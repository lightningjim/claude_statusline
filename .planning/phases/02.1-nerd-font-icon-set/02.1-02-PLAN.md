<objective>
Convert the weather conditions block to Nerd Font glyphs at full `wi-*` granularity (D-03):
distinct day/night glyphs and distinct glyphs for snow / sleet / freezing-rain /
rain-snow-mix / thunderstorm / thunderstorm-with-rain / fog-haze / windy, with live moon
phase on clear/mostly-clear nights (D-04) and semantic ANSI color (D-08). Retain the Phase 2
emoji table as the `icon_set="emoji"` fallback (D-06).

This is the meteorologist-fidelity payoff of the phase. It also makes a deliberate
architectural change: the condition icon is currently resolved to a glyph string at FETCH
time and baked into `cache.json` (run_refresh -> fetch_weather -> _icon_to_emoji -> stored as
`weather["icon"]`). To honor D-04 (moon phase is time-dependent — a baked glyph would show a
stale moon) and D-08 (color resolved with intent at render) and D-07 (toggle takes effect
immediately), the fetch path is changed to store the RAW NWS tokens (textDescription +
icon URL), and the render path (`_weather_segment`) resolves glyph + color via the new
`_icon_to_glyph`. Alert severity coloring (D2-11) is preserved untouched.

Implements ENH-04 weather/alert segment per D-03, D-04, D-05, D-06, D-08.
Purpose: a forecaster's read of the sky — real moon phase, freezing rain ≠ rain.
Output: dual condition tables, the render-time resolver, the cache token migration.
</objective>

|----------|-------------|
| NWS API -> cache.json | Third-party JSON; textDescription + icon URL now stored as raw strings and later substring-matched. Already-trusted path (Phase 2). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02.1-04 | Tampering / Injection | NWS textDescription/icon stored raw then rendered | mitigate | Tokens are only substring-matched against a fixed keyword table to SELECT a developer-authored static glyph constant; the NWS string itself is never emitted as the glyph, so a malicious description cannot inject ANSI escapes via the condition glyph. Temp/PoP remain numeric-coerced as in Phase 2. |
| T-02.1-05 | Denial of Service | resolver / moon computation failure on render path | mitigate | _icon_to_glyph and _weather_segment wrap bodies in try/except -> omit conditions chunk / fall back to sun event (D2-12); moon-index helper clamps; render never raises. |
| T-02.1-06 | Tampering | semantic color category mapping | accept | _wx_color maps to static ANSI constants; unknown category -> RESET. No untrusted data in the escape strings. |

No high-severity threats introduced: no NEW network calls (fetch path unchanged in behavior,
only its stored payload shape changed), no new untrusted parsing surface beyond the
already-fetched-and-trusted NWS fields, and the rendered glyph is always a static constant
rather than echoed NWS text.
</threat_model>
