<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Claude Code → script stdin | Semi-trusted local process pipes one JSON object; payload may be malformed, empty, or pathologically large |
| install.py → ~/.claude/settings.json | Installer rewrites a user-owned config file that may already contain unrelated keys |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Denial of Service | stdin parse in claude-statusline.py | mitigate | Parse with `json.load` only (never `eval`); wrap in try/except; no unbounded loops over input — single pass field access so a large/pathological payload cannot hang the bar (RUN-02) |
| T-01-02 | Tampering / Elevation | malformed stdin | mitigate | Never `eval`/`exec` stdin; only `json.load`. Malformed → minimal safe line + exit 0 (D-11), no code path executes input |
| T-01-03 | Tampering | install.py writing settings.json | mitigate | Parse-merge-backup: read+`json.load`, write `settings.json.bak` first, preserve existing keys, refuse to overwrite on parse error (D-14, security_context); idempotent |
| T-01-SC | Tampering | npm/pip/cargo installs | mitigate | N/A — Phase 1 installs zero third-party packages; pyproject `dependencies = []`. No package legitimacy audit required. ASVS L1 satisfied for supply chain |

No `high` severity threats: no network, no secrets, no shell-out, no third-party deps in Phase 1. ASVS L1 is satisfied for this local single-file CLI.
</threat_model>
