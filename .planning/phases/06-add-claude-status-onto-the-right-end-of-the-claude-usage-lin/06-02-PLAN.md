<objective>
Wire the Claude service-health indicator onto the RENDER path: a new
`_claude_status_segment(data, cfg)` builder that reads the `claude_status` cache
section (Plan 01's contract), applies quiet-when-healthy / cold-cache omission
(D-01), resolves the severity color + icon_set glyph and ANSI-sanitizes the
incident title (D-03/D-04), and is appended to the bottom line AFTER the weekly
segment with the existing 3-space separator (D-06). Includes the render-path
spawn trigger so status refreshes even when weather is disabled, plus the full
render + E2E + ANSI-injection test set.

Purpose: Make the indicator visible (and silent when it should be), consuming the
data contract from Plan 01 without inline network work.

Output: `_claude_status_segment`, the render_bottom_line append, a render-path
`maybe_spawn_refresh` call, and render/E2E tests in tests/test_claude_status.py.
</objective>
