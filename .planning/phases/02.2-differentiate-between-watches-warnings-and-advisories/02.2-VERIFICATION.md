### Gap Closure Evidence

#### WR-01 — Statement/Other intensity axis (CLOSED, commit 14a7487)

**Root cause:** `"Statement/Other": RESET` in `_alert_color`'s hue_map. `RESET` (`\x1b[0m`) cancels all prior SGR attributes including a prepended BOLD or DIM.

**Fix verified:**
- `DEFAULT_FG = "\033[39m"` added at `/home/kcreasey/Documents/Projects/claude_statusline/claude-statusline.py` line 83, with comment: "default foreground only — neutral hue that preserves BOLD/DIM (unlike RESET, which cancels intensity); see _alert_color (WR-01/D-06)".
- `_alert_color` docstring updated to document DEFAULT_FG as the Statement/Other hue and explain why RESET was wrong.
- `hue_map["Statement/Other"]` at L1199 is now `DEFAULT_FG`, not `RESET`.
- Test `TestAlertColor.test_statement_intensity_band_survives` at line 602 in `tests/test_weather_alerts.py`: creates Immediate+Observed and Future+Possible Statement alerts, asserts BOLD in immediate result, DIM in future result, and the two results are not equal. Passes.

#### WR-02 — All-control-char event sanitizes to hollow glyph+space (CLOSED, commit 14a7487)

**Root cause:** `detail = f"{class_glyph} {safe_event}"` with no guard for `safe_event == ""` after sanitization.

**Fix verified:**
- Sanitize block at L2604-2612 in `_weather_segment`: the comprehension now chains `.strip()` on the slice result (`[...].strip()`). Then an explicit guard: `if not safe_event: safe_event = best_class`. Comment: "D-10: never emit a hollow glyph — if sanitization left nothing (e.g. an all-control-char event), fall back to the class name (WR-02)".
- Test `TestWeatherSegmentAlertOverrideV2.test_all_control_char_event_falls_back_to_class_name` at line 1382 in `tests/test_weather_alerts.py`: event `"\x01\x02\x03\x04"` with VTEC `SV.W` (Warning class); asserts "Warning" in rendered output, no `warn_glyph + " " + RESET` hollow pattern, no raw control bytes. The test is `_WEATHER_OK`-gated and skipped in the base env (consistent with other V2 tests) — code correctness and test presence verified. Passes.
