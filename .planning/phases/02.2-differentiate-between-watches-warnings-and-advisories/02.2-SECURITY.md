## Trust Boundaries

| Boundary | Description | Data Crossing |
|----------|-------------|---------------|
| NWS cache → alert helpers | Alert dicts (`event`, `VTEC`, `urgency`, `certainty`, `severity`) originate from api.weather.gov via the Phase 2 fetch and are parsed by the classification/selection layer | Untrusted third-party JSON fields |
| NWS cache → rendered top line (stdout) | The alert `event` name crosses from untrusted api.weather.gov data into the terminal — the only place untrusted text reaches stdout in this phase | Untrusted text → ANSI terminal |

## Threat Register

| Threat ID | Category | Component | Disposition | Mitigation | Status |
|-----------|----------|-----------|-------------|------------|--------|
| T-02.2-01 | Denial of Service | `_classify_alert_class`, `_alert_intensity`, `_alert_color`, `select_alert`, `_build_alert_tally` parsing malformed/missing fields | mitigate | Each helper wraps its body in `try/except Exception` returning a safe default (`"Statement/Other"`, `""`, `YELLOW`, `(None,[])`). Never-raise tests: `test_never_raises_loop`, `test_never_raises_on_malformed`, `test_tolerates_unknown_severity`, `test_malformed_alert_does_not_raise` | closed |
| T-02.2-02 | Tampering | Crafted VTEC string (<5 fields, non-letter significance) | mitigate | `if len(fields) >= 5:` guard before indexing field 4; unknown significance misses `_VTEC_SIG_TO_CLASS.get()` → event-name fallback → `"Statement/Other"`. `test_vtec_with_fewer_than_5_fields_falls_through_to_event_name` confirms no index error | closed |
| T-02.2-03 | Information Disclosure / Spoofing | NWS `event` text (Plan 01 scope) | accept | Plan 01 layer never renders raw `event` to stdout — only uses it as a classification predicate (`event.endswith(...)`). Rendering + sanitization deferred to T-02.2-04 (Plan 02). Verified Plan-01 commits do not touch any render/print path | closed |
| T-02.2-04 | Tampering / Information Disclosure | NWS `event` text rendered into the trailing detail | mitigate | Inline sanitizer keeps only `ch == " " or (ch.isprintable() and ch != "\x1b")`, truncates `[:64]`, then `.strip()`; empty-after-strip falls back to class name (not a hollow glyph). `test_event_text_sanitized_no_esc_byte` + `test_all_control_char_event_falls_back_to_class_name` | closed |
| T-02.2-05 | Denial of Service | Unbounded event length / many active alerts blowing up the width-unbounded top line | mitigate | `event` truncated to 64 chars; remainder rendered as a compact grouped per-class glyph+count via `_build_alert_tally` (max 4 classes), not per-alert | closed |
| T-02.2-06 | Denial of Service | Malformed alert dict in the `_weather_segment` override block | mitigate | Outer `try/except Exception: pass` wraps the entire alert-override block → falls through to `_sun_segment`; all four helpers are independently never-raise. `test_malformed_alert_override_falls_through_to_sun` | closed |
| T-02.2-SC | Tampering (supply chain) | npm/pip/cargo installs | n/a | No new package installs — in-repo Python only. `pyproject.toml` unmodified across the phase; deps remain `requests` + `astral`. Package-legitimacy gate not applicable | closed |

**Notable verification:** the gap-closure commit `14a7487` added `.strip()` to the event sanitizer. Operator order is filter → truncate → strip, so the unconditional `\x1b`/non-printable exclusion in the generator executes *before* `.strip()` — ESC/control-char stripping is intact, not weakened. The empty-event fallback emits `best_class` (internal ASCII classification output), never raw NWS data.
