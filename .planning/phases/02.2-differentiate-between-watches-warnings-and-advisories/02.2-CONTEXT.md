# Phase 02.2: Differentiate between watches, warnings, and advisories - Context

**Gathered:** 2026-06-06
**Status:** Ready for planning

<domain>
## Phase Boundary

Make the top-line alert detail distinguish the three NWS hazard classes — **Warning** (hazard occurring/imminent → act now), **Watch** (conditions favorable → be prepared), **Advisory** (less serious → be aware) — as a dimension **distinct from CAP `severity`**, so the bar tells you *what kind of action* an active alert demands at a glance. A fourth **Statement/Other** bucket covers non-W/A/Y products.

This **extends D2-11** (Phase 2's severity-only alert override) within the existing `_weather_segment` → `select_alert` → `_alert_color` path. The trailing detail is still the alert override that replaces the sun event when an alert is active (WX-04).

**In scope:** classifying each alert by hazard class; a distinct glyph per class; class-driven color; folding NWS `urgency`/`certainty` into color intensity and selection ranking; class-first primary selection; a per-class tally for the "+N" remainder.

**Out of scope:** new weather variables (dewpoint/wind/apparent-temp — v2), multi-location (ENH-03), changing the conditions/precip chunks, the bottom line, and anything from Phases 1/3/4/5. No new network calls — alerts already arrive via the Phase 2 fire-and-forget fetch/cache.

</domain>

<decisions>
## Implementation Decisions

### Classification (how an alert's class is determined)
- **D-01:** Classify by **VTEC significance code** as the authoritative source: parse `properties.parameters.VTEC` and read the significance letter — **W** = Warning, **A** = Watch, **Y** = Advisory, **S** = Statement (also F/O/N → Statement/Other bucket).
- **D-02:** **Event-name fallback** when no VTEC is present (non-VTEC products: Air Quality Alert, Special Weather Statement, Red Flag Warning issued without VTEC, etc.) — classify by the trailing word of `properties.event` (Warning / Watch / Advisory). Anything that matches none → **Statement/Other**.
- **D-03:** Four classes total: **Warning, Watch, Advisory, Statement/Other**. The classifier must never raise — unclassifiable → Statement/Other (omit-not-fake discipline, D-10).

### Visual distinction
- **D-04:** **Distinct glyph per class**, replacing the single `⚠`. Nerd Font glyph primary, emoji fallback — mirroring the Phase 02.1 `icon_set` toggle and its installed-font **cmap guard** (every new glyph MUST be validated against the JetBrains Nerd Font cmap, as `test_nerd_icons.py` enforces). Glyph choice is Claude's discretion within that constraint, but must read as warning/watch/advisory at a glance.
- **D-05:** **Class-driven hue** (replaces D2-11's severity-driven alert color): **Warning = red, Watch = yellow, Advisory = cyan**, Statement/Other = a neutral/visible default. Hue encodes the action level, decoupled from noisy/inconsistent CAP severity.
- **D-06:** **Urgency + certainty modulate brightness (intensity axis)** on top of the class hue: **Immediate + Observed → bold/bright; Expected / Likely → normal; Future / Possible → dim** (via ANSI bold + DIM). Hue still tells you the class; brightness tells you immediacy. Exact threshold mapping for the three bands is Claude's discretion, but the three-band bold→normal→dim model is locked.

### Selection & multi-alert display
- **D-07:** **Class-first primary selection** (replaces severity-only `select_alert`): rank **Warning > Watch > Advisory > Statement/Other**, then break ties by the composite **severity → urgency → certainty**. A Severe Thunderstorm *Warning* outranks a same-severity Tornado *Watch*.
- **D-08:** **Per-class tally** for the remainder instead of a flat `+N`: group the other active alerts by class and show counts beside each class glyph. Must stay width-conscious (the top line has no guaranteed width); collapse/abbreviate when crowded is Claude's discretion, but the tally is grouped by class, not a single count.

### Claude's Discretion
- Exact per-class glyph codepoints (nerd) and emoji fallbacks — subject to the cmap guard.
- Precise urgency/certainty → {bold, normal, dim} threshold table.
- VTEC parsing mechanism (the significance letter is the 5th `.`-delimited field of each VTEC string; pick the most relevant VTEC when several are present).
- Exact per-class tally rendering/abbreviation under width pressure.
- Whether to strip the now-partly-redundant class word from the event text (kept open; the user did not pick the "drop class word" option, so default is to retain the full NWS event name).

</decisions>

<specifics>
## Specific Ideas

- **User is a degreed meteorologist** — correct W/W/A semantics and high NWS data fidelity matter. Warning = act now, Watch = be prepared, Advisory = be aware; these are real operational distinctions, not severity synonyms.
- The user deliberately chose the **richer** options (full urgency/certainty fold-in, per-class tally) over the minimal ones — favor signal/fidelity over terseness here, within the top line's width reality.
- Target shape (illustrative — glyphs/exact rendering TBD in planning):
  - Single warning: `[… | <warn-glyph> Tornado Warning]` (red, bold for Immediate+Observed)
  - Watch primary with others active: `[… | <watch-glyph> Tornado Watch  <warn-glyph>1 <adv-glyph>2]` (watch=yellow; tally per class)
  - Advisory only: `[… | <adv-glyph> Wind Advisory]` (cyan, dim if Future/Possible)

</specifics>
