<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| stdin (session JSON) → git subprocess | `workspace.current_dir` / `cwd` is untrusted input that becomes the `-C <dir>` argument to git |
| local repo working tree → render path | a pathological/huge repo can make `git status` slow |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering / EoP | `_run_git` argv built from stdin-derived `cwd` | mitigate | Fixed argv list `["git","-C",cwd,*args]` passed to `subprocess.run` WITHOUT `shell=True`; `cwd` is only ever a positional arg to `-C`, never concatenated into a shell string. Verified by an assertion that the source contains no `shell=True`. |
| T-04-02 | Denial of Service | `_run_git` against a pathological repo (huge dirty tree) | mitigate | `subprocess.run(timeout=0.15)` bounds runtime; on TimeoutExpired the child is killed and the except returns None. Tested with an artificially tiny timeout asserting None (not a hang). |
| T-04-03 | Denial of Service | git binary absent / missing dir | mitigate | Blanket `except Exception` in `_run_git` catches FileNotFoundError/OSError → returns None; no pre-probe needed. Tested against a non-existent directory. |
| T-04-04 | Tampering (output spoofing) | branch name containing ANSI/control chars leaking into the bar | accept | LOW risk for a single-user personal tool; branch labels are printed inert, never re-executed. Optional control-char stripping deferred (not required by CONTEXT). No PII, no network. |
</threat_model>
