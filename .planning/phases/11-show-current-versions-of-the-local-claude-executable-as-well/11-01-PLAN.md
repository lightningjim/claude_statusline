<objective>
Implement the version-display fragment end to end inside the single-file statusline:
source the Claude version from the stdin `version` field and the GSD version from the
installed-plugins ledger, render them as a dimmed trailing fragment on the bottom line
(after the Claude-status segment), gated by a new `show_versions` toggle and honoring the
existing `icon_set` nerd/text fallback — with omit-not-fake on every bad-data path.

Purpose: surface the running Claude Code version and the active GSD plugin version
unobtrusively, without any subprocess or network call (honors the never-block contract).
Output: new reader (`_read_installed_gsd_version`), sanitizer (`_sanitize_version`), builder
(`_versions_fragment`), two `_NF_VERSION_*` glyph constants, a `display.show_versions`
default, render_bottom_line wiring, and a new `tests/test_versions_fragment.py` module.

This phase predates formal v1.1 REQ-IDs (consistent with prior CONTEXT-driven phases). Five
phase-local `VER-` requirement IDs are derived for traceability, mapped to CONTEXT decisions:
- VER-01 = Claude version from stdin `version`, omit-not-fake (D-03, D-04)
- VER-02 = GSD version from installed_plugins.json ledger array `[0]["version"]`, always-show, omit-not-fake (D-05, D-06, D-07)
- VER-03 = dimmed trailing fragment after Claude-status segment, single-space internal join, None-filtered (D-01, D-02, D-09)
- VER-04 = Nerd Font glyphs with text-label fallback per icon_set (D-08, D-11)
- VER-05 = show_versions config toggle, default True (D-10)
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Claude Code -> stdin `version` -> terminal | The `version` string is piped in and echoed into the ANSI bottom line; untrusted-content rendering boundary |
| local filesystem -> installed_plugins.json `version` -> terminal | The ledger version string is read from a home-path file and echoed into the line; untrusted-content + file-read boundary |
| home filesystem -> _read_installed_gsd_version | Local JSON read from a user-home path (size/parse/shape) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-01 | Tampering/Injection | _versions_fragment rendering of stdin `version` + ledger `version` | mitigate | `_sanitize_version` rejects any value with ESC/control/ANSI chars or >64 len; only `[0-9A-Za-z._+-]`-class version-like strings render, else the piece is omitted (omit-not-fake). No raw escape from either echoed value can reach the terminal. Covered by an explicit injection test. |
| T-11-02 | Denial of Service | _read_installed_gsd_version reading installed_plugins.json | mitigate | Byte-capped read `fh.read(_GSD_MAX_BYTES)` (65 536) before json.loads bounds memory/CPU on a huge or adversarial ledger; whole body in try/except -> None. |
| T-11-03 | Denial of Service / never-crash | render_bottom_line + builder on malformed JSON / non-list entry | mitigate | Step-by-step isinstance guards (dict, dict, non-empty list, dict, non-empty str) before any index; builder body and render_bottom_line body each wrapped in `try/except Exception: return None`. No exception escapes the bar. |
| T-11-04 | Information Disclosure / path | expanduser of ~/.claude/plugins/installed_plugins.json | accept | Reads a single fixed, well-known ledger path under the user's own home; no user-controlled path component, no symlink-following beyond normal home resolution. Low risk; no PII echoed (only a version number). |
| T-11-SC | Tampering (supply chain) | npm/pip/cargo installs | accept | This phase adds NO new packages or dependencies — all code is stdlib + existing module. No install step exists, so the package-legitimacy gate is N/A this phase. |
