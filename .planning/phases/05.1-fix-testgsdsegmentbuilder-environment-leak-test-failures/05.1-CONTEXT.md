# Phase 05.1: Fix TestGsdSegmentBuilder environment-leak test failures - Context

**Gathered:** 2026-05-29
**Status:** Ready for planning

<domain>
## Phase Boundary

Make the 9 failing `TestGsdSegmentBuilder` tests in `tests/test_gsd_segment.py`
hermetic. The failures are **not** a product bug — `_gsd_segment` is correct.
They fail because of a **wall-clock dependency** in the test fixtures.

**Root cause (confirmed):** The shared fixtures `_VALID_HANDOFF`,
`_VERIFYING_HANDOFF`, and `_BLOCKED_HANDOFF` carry a hardcoded timestamp
`2026-05-29T20:00:00.000Z`. The source (`claude-statusline.py`
`_infer_gsd_lifecycle`, line ~1623) treats a HANDOFF as *live* only when its
timestamp is within `_GSD_HANDOFF_STALE_SECONDS = 3600` (1 hour) of `now`.
`TestGsdSegmentBuilder._call` feeds those fixtures through a monkeypatched
`_read_gsd_state` **without refreshing the timestamp**, and most builder cases
pass `roadmap=None`/`""`. So once the suite runs >1 hour after that fixed
instant, the handoffs read as stale → roadmap fallback → empty roadmap →
`_gsd_segment` returns `None` → 9 `assertIsNotNone` assertions fail.

The sibling class `TestInferGsdLifecycle` already avoids this by stamping
`datetime.now(timezone.utc)` into its fixtures via `_make_live_state()`.

**Scope anchor:** Fix lives entirely in `tests/test_gsd_segment.py`. Source code
is correct and is NOT modified.

**Out of scope:** The pre-existing module-level `tempfile.mkdtemp()` directories
that are never cleaned up (across ~6 test files, including this file's
`_E2E_HOME`) — real but separate test-hygiene concern, deferred. No changes to
`claude-statusline.py`. No broader audit of other test files.

</domain>

<decisions>
## Implementation Decisions

### Fix mechanism
- **D-01:** **Refresh the handoff timestamp to `now` at call time.** In
  `TestGsdSegmentBuilder._call` (or a tiny local helper it uses), copy the
  handoff dict and overwrite `timestamp` with
  `datetime.now(timezone.utc).strftime("%Y-%m-%dT%H:%M:%S.000Z")` before passing
  it into the monkeypatched `_read_gsd_state`. This mirrors the existing,
  proven `TestInferGsdLifecycle._make_live_state()` pattern. The tests continue
  to exercise the **real** staleness logic — they just stop depending on the
  ambient wall clock.
  - **NOT chosen:** freezing/injecting a fixed clock. The source calls
    `datetime.now()` inline with no injection seam; monkeypatching the module's
    clock is heavier than warranted for this fix and would touch more surface.

### Scope
- **D-02:** **Fix only the 9 failing `TestGsdSegmentBuilder` tests.** Touch
  `tests/test_gsd_segment.py` only. Do not sweep other test files, do not touch
  the `mkdtemp` cleanup issue, do not modify source. Matches the phase title
  exactly and keeps the change auditable.
  - The 9 affected tests (all currently `AssertionError: unexpectedly None`):
    `test_executing_contains_plan_id`, `test_executing_contains_green`,
    `test_executing_plan_id_not_wrapped_in_color`,
    `test_executing_contains_task_progress`, `test_blocked_contains_red`,
    `test_verifying_contains_yellow`, `test_output_is_bracketed`,
    `test_icon_set_emoji_uses_fallback_glyphs`,
    `test_icon_set_emoji_output_nonempty`.
  - Idle/done/None cases in the same class already pass (they use
    `_NULL_HANDOFF` + roadmap fixtures or assert `None`) and must stay passing.

### Regression guard
- **D-03:** **Add an explicit guard test** that asserts the builder resolves a
  freshly-stamped (live) handoff to a non-`None`, executing segment regardless
  of wall-clock time — so reintroducing a fixed/stale timestamp fails loudly
  and obviously, rather than silently rotting after an hour. Name it for intent
  (e.g. `test_builder_live_at_any_walltime`).

### Claude's Discretion
- Whether D-01's refresh lives inline in `_call` vs. a small private helper in
  the test module — planner/executor's call. (Kyle chose "just the 9 tests" for
  scope, not the cross-class shared-helper refactor, so keep it local to this
  file; do not refactor `_make_live_state` itself.)
- Exact name/wording of the D-03 guard test, as long as intent is unmistakable.
- Whether to add a short explanatory comment at the fixture definitions noting
  that timestamps must be stamped at call time (encouraged, not required).

</decisions>

<specifics>
## Specific Ideas

- The fix must keep tests exercising the genuine staleness path (D-05 from
  Phase 05), not bypass it. "Refresh to now" is preferred precisely because it
  keeps the real `handoff_live` age check in play.
- Mirror the established `_make_live_state()` shape so the two test classes read
  consistently, even though we're not extracting a shared helper.

</specifics>

<code_context>
## Existing Code Insights

### Reusable Assets
- `TestInferGsdLifecycle._make_live_state()` (`tests/test_gsd_segment.py`
  ~419–442): the exact timestamp-refresh idiom to mirror —
  `datetime.now(timezone.utc).strftime("%Y-%m-%dT%H:%M:%S.000Z")` applied to a
  copied handoff dict. Reuse the technique inline in `_call` (do not extract a
  shared helper per D-02 scope).

### Established Patterns
- Builder tests monkeypatch `_read_gsd_state` and restore it in a `finally`
  block (`_call`, ~842–862). The timestamp refresh must happen on the dict the
  fake reader returns, before `_infer_gsd_lifecycle` runs.
- Fixtures are module-level dicts shared across cases — copy with `dict(...)`
  before mutating (already the convention in `_make_live_state` and several
  `TestInferGsdLifecycle` methods) to avoid cross-test mutation leaks.

### Integration Points
- `_call(handoff, state_fm, roadmap, cfg_override)` → fake `_read_gsd_state` →
  `_gsd_segment(data, cfg)` → `_infer_gsd_lifecycle(state)`. The timestamp is
  consumed at the `_infer_gsd_lifecycle` staleness check; refresh upstream of it.

</code_context>
