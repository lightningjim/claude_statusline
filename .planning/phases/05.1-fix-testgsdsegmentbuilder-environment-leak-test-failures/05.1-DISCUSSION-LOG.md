## Pre-discussion diagnosis (presented to user)

Confirmed via running the suite + reading source:
- 9 `TestGsdSegmentBuilder` tests fail with `AssertionError: unexpectedly None`.
- Cause: fixtures `_VALID_HANDOFF` / `_VERIFYING_HANDOFF` / `_BLOCKED_HANDOFF`
  carry a hardcoded `timestamp` `2026-05-29T20:00:00.000Z`; source treats a
  HANDOFF as live only within `_GSD_HANDOFF_STALE_SECONDS = 3600` (1h) of now.
  ~3h after that instant the handoffs read stale → roadmap fallback → empty
  roadmap → `None`.
- Sibling class `TestInferGsdLifecycle` is immune because `_make_live_state()`
  stamps `datetime.now()`.
- Source is correct; fix is test-only.

## Questions Asked

### Fix mechanism
- **Options presented:**
  - Refresh timestamp to now (stamp `datetime.now()` in `_call`/helper; mirrors
    `_make_live_state`; no source change) — *recommended*
  - Freeze the clock (monkeypatch module time source; needs a seam; heavier)
  - Shared live-handoff helper (promote now-stamping to one module-level helper
    used by both classes)
- **User selected:** **Refresh timestamp to now.**

### Scope
- **Options presented:**

  - Also sweep time-dependent fixtures across the suite
- **User selected:** **Just the 9 failing tests.**

### Regression guard
- **Options presented:**
  - Just fix it (+ comment)
  - Helper + convention
  - Add a guard test (asserts a live handoff yields non-None executing segment
    at any wall-clock time)
- **User selected:** **Add a guard test.**

## Resulting decisions
- D-01: Refresh handoff timestamp to `now` at call time in `_call`.
- D-02: Scope limited to the 9 failing `TestGsdSegmentBuilder` tests, this file
  only; no source changes; `mkdtemp` leaks deferred.
- D-03: Add an explicit wall-clock-independent guard test.

- Cross-class shared live-handoff helper — set aside per minimal-scope choice.
