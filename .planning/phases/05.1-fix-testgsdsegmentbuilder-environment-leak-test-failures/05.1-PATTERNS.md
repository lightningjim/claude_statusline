### 1. The analog to MIRROR — `TestInferGsdLifecycle._make_live_state()`

**Analog:** `tests/test_gsd_segment.py` lines 419-442 (sibling class, same file).

This is the proven, in-tree idiom the fix copies. It refreshes a fixture's
`timestamp` to wall-clock `now` so the real staleness check reads it as live.
The technique to lift: `dict(_VALID_HANDOFF)` copy + overwrite `timestamp` with
`datetime.now(timezone.utc).strftime("%Y-%m-%dT%H:%M:%S.000Z")`.

**Full method excerpt** (lines 419-442):
```python
def _make_live_state(self, handoff_override=None, state_override=None, roadmap=None):
    """Build a live state dict with a recent timestamp."""
    from datetime import datetime, timezone
    now_z = datetime.now(timezone.utc).strftime("%Y-%m-%dT%H:%M:%S.000Z")
    handoff = dict(_VALID_HANDOFF)
    handoff["timestamp"] = now_z
    if handoff_override:
        handoff.update(handoff_override)
    state_fm = {
        "milestone": "v1.0",
        "status": "executing",
        "progress": {
            "total_plans": 15,
            "completed_plans": 13,
            "percent": 71,
        },
    }
    if state_override:
        state_fm.update(state_override)
    return {
        "handoff": handoff,
        "state": state_fm,
        "roadmap": roadmap or _ROADMAP_WITH_INCOMPLETE,
    }
```

**Key idiom to copy (3 lines):**
```python
from datetime import datetime, timezone
now_z = datetime.now(timezone.utc).strftime("%Y-%m-%dT%H:%M:%S.000Z")
handoff = dict(_VALID_HANDOFF)   # copy before mutating — avoid cross-test leak

```

### 2. The method to FIX — `TestGsdSegmentBuilder._call()`

**Target:** `tests/test_gsd_segment.py` lines 819-862. The monkeypatched
`fake_read_gsd_state` returns the raw fixture `handoff` dict **without
refreshing its timestamp** (lines 833-840). That stale timestamp flows into
`_infer_gsd_lifecycle` and ages past the 1-hour window → `handoff_live = False`
→ roadmap fallback → empty roadmap → `None`.

**Current method excerpt** (lines 819-862):
```python
def _call(self, handoff=None, state_fm=None, roadmap=None, cfg_override=None):
    """Call _gsd_segment with monkeypatched _read_gsd_state.

    Passes a fake project_dir that has a .planning/ directory stub (so the
    os.path.isdir check passes) but monkeypatches _read_gsd_state to return
    controlled fixture data.
    """
    import tempfile as _tmpfile, os as _os

    # We need an actual directory for the os.path.isdir(.planning) guard
    with _tmpfile.TemporaryDirectory() as tmpdir:
        planning_dir = _os.path.join(tmpdir, ".planning")
        _os.makedirs(planning_dir)

        def fake_read_gsd_state(_planning_dir):
            if handoff is None:
                return None
            return {
                "handoff": handoff,            # <-- raw fixture, STALE timestamp
                "state": state_fm or {},
                "roadmap": roadmap or "",
            }

        original = self.mod._read_gsd_state
        self.mod._read_gsd_state = fake_read_gsd_state
        try:
            cfg = {
                "display": {"icon_set": "nerd", "show_gsd": True, "bar_style": "shade"},
                "toggles": {"show_thinking_glyph": True},
                "thresholds": {"warn": 70, "crit": 90},
            }
            if cfg_override:
                for k, v in cfg_override.items():
                    if isinstance(v, dict) and isinstance(cfg.get(k), dict):
                        cfg[k].update(v)
                    else:
                        cfg[k] = v
            data = {
                "workspace": {"current_dir": tmpdir, "project_dir": tmpdir},
                "cwd": tmpdir,
            }
            return self.mod._gsd_segment(data, cfg)
        finally:
            self.mod._read_gsd_state = original
```

**EXACT insertion point (D-01):** Refresh the timestamp on a COPY of the
`handoff` dict before `fake_read_gsd_state` returns it. The cleanest seam is
inside `fake_read_gsd_state`, after the `if handoff is None` guard and before
the `return {...}` — so the `None`/idle paths are untouched and every live
handoff is stamped to `now`:

```python
def fake_read_gsd_state(_planning_dir):
    if handoff is None:
        return None
    # D-01: stamp the handoff timestamp to NOW so the real staleness
    # check (_GSD_HANDOFF_STALE_SECONDS) reads it as live, independent of
    # the ambient wall clock. Mirrors TestInferGsdLifecycle._make_live_state.
    from datetime import datetime, timezone
    live_handoff = dict(handoff)
    live_handoff["timestamp"] = datetime.now(timezone.utc).strftime("%Y-%m-%dT%H:%M:%S.000Z")
    return {
        "handoff": live_handoff,
        "state": state_fm or {},
        "roadmap": roadmap or "",
    }
```

> Why a copy (`dict(handoff)`): fixtures `_VALID_HANDOFF` / `_BLOCKED_HANDOFF` /
> `_VERIFYING_HANDOFF` are module-level and shared across tests; mutating them in
> place would leak across cases. Copying is the established convention
> (`_make_live_state` line 423, and `test_zero_total_tasks_*`/`test_malicious_*`
> at lines 981, 996 all do `dict(_VALID_HANDOFF)` first).

> Note: `_NULL_HANDOFF` (idle) cases still get a fresh timestamp here, which is
> harmless — those tests assert the roadmap-fallback/idle path which keys off the
> null `plan` field, not on liveness. The `handoff is None` early-return path
> (`test_read_gsd_state_returns_none_gives_none`) is left exactly as-is.

**Suggested guard test (D-03), modeled on the above** (exact name at executor's
discretion — intent must be unmistakable):
```python
def test_builder_live_at_any_walltime(self):
    """D-03 guard: a freshly-stamped (live) handoff resolves to a non-None
    executing segment regardless of the ambient wall clock. Reintroducing a
    fixed/stale fixture timestamp must fail this loudly, not rot silently
    after _GSD_HANDOFF_STALE_SECONDS.

    Note: this asserts the post-fix _call ALWAYS stamps the handoff to now,
    so the real staleness check (_infer_gsd_lifecycle) sees it as live.
    """
    result = self._call(handoff=_VALID_HANDOFF)
    self.assertIsNotNone(result, "Live handoff must resolve to a segment (D-03 guard)")
    self.assertIn("05-02", result)        # executing plan id present
    self.assertIn("\033[32m", result)     # GREEN executing glyph → confirms 'live/executing'
```

> The GREEN-code assertion (`\033[32m`) is the load-bearing part: it confirms the
> handoff resolved as a LIVE executing state rather than the DIM idle fallback
> (`\033[2m`) that the stale-timestamp bug produced. This is what makes the guard
> distinguish "live" from "stale-fell-back-to-idle".

## Shared Patterns

### Copy-before-mutate (module-level fixture safety)
**Source:** `tests/test_gsd_segment.py` line 423 (`_make_live_state`), lines 981 & 996.
**Apply to:** Every place a fixture handoff is restamped.
```python
handoff = dict(_VALID_HANDOFF)   # never mutate the shared module-level dict
handoff["timestamp"] = ...
```

### Timestamp-refresh idiom (the canonical "make live" line)
**Source:** `tests/test_gsd_segment.py` lines 421-424.
**Apply to:** `_call`'s `fake_read_gsd_state`, and the D-03 guard if it stamps inline.
```python
from datetime import datetime, timezone
now_z = datetime.now(timezone.utc).strftime("%Y-%m-%dT%H:%M:%S.000Z")
```
The `"%Y-%m-%dT%H:%M:%S.000Z"` format matters: the source parses it via
`datetime.fromisoformat(ts_raw.replace("Z", "+00:00"))` (claude-statusline.py
line 1622), so the trailing `Z` and millisecond `.000` must be preserved.

### Monkeypatch-with-finally-restore
**Source:** `tests/test_gsd_segment.py` lines 842-862.
**Apply to:** Unchanged — preserve the existing `original`/`try`/`finally`
restore of `self.mod._read_gsd_state`. The fix lives INSIDE `fake_read_gsd_state`,
not in the patch/restore mechanics.

# --- Check HANDOFF liveness (D-05) ---
handoff_plan = handoff.get("plan")
handoff_live = False
if isinstance(handoff_plan, str) and handoff_plan:
    ts_raw = handoff.get("timestamp") or ""
    try:
        ts = datetime.fromisoformat(ts_raw.replace("Z", "+00:00"))
        age = (datetime.now(timezone.utc) - ts).total_seconds()
        handoff_live = age <= _GSD_HANDOFF_STALE_SECONDS
    except Exception:
        handoff_live = False  # bad timestamp → treat as stale

flips the builder onto the empty-roadmap → `None` path. The fix keeps this real
check in play (D-01 / phase D-05) — it just stops feeding it a wall-clock-aged
timestamp.
