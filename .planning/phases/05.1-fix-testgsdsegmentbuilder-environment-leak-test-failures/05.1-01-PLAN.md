<objective>
Make the 9 failing `TestGsdSegmentBuilder` tests in `tests/test_gsd_segment.py`
hermetic by refreshing the handoff timestamp to `now` at call time, then add a
regression guard test that fails loudly if a fixed/stale timestamp is ever
reintroduced.

Purpose: The failures are NOT a product bug — `_gsd_segment` /
`_infer_gsd_lifecycle` are correct. The shared fixtures carry a hardcoded
timestamp (`2026-05-29T20:00:00.000Z`) that ages past the 1-hour staleness
window (`_GSD_HANDOFF_STALE_SECONDS = 3600`). Once the suite runs >1 hour after
that instant, the handoffs read as stale → roadmap fallback → empty roadmap →
`_gsd_segment` returns `None` → 9 `assertIsNotNone` assertions fail. The fix
keeps the REAL staleness path live (per D-01 / phase-05 D-05); it just stops
feeding the check a wall-clock-aged timestamp.

Output: `tests/test_gsd_segment.py` modified (timestamp refresh in `_call` +
one new guard test). No source change.
</objective>
