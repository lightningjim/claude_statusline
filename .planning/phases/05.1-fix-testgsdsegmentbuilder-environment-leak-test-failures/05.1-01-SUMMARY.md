# Phase 05.1 Plan 01: Fix TestGsdSegmentBuilder Environment-Leak Test Failures Summary

**One-liner:** Timestamp refresh via dict(handoff) copy in fake_read_gsd_state makes 9 TestGsdSegmentBuilder tests wall-clock-independent, plus a D-03 GREEN-code guard test prevents silent regression.

## What Was Built

Task 1 fixed `TestGsdSegmentBuilder._call`'s inner `fake_read_gsd_state` by adding a copy-before-mutate timestamp refresh after the `if handoff is None` guard. The raw fixture dict is copied via `dict(handoff)` and its `timestamp` key is overwritten with `datetime.now(timezone.utc).strftime("%Y-%m-%dT%H:%M:%S.000Z")` — the exact format the source parser expects (`datetime.fromisoformat(ts_raw.replace("Z", "+00:00"))`). This keeps the real staleness logic (`_GSD_HANDOFF_STALE_SECONDS = 3600`) exercised while stopping the tests from feeding it a wall-clock-aged fixture timestamp.

An optional explanatory comment was also added at the fixture block noting that timestamps must be restamped at call time.

Task 2 added `test_builder_live_at_any_walltime` to `TestGsdSegmentBuilder`. It calls `_call(handoff=_VALID_HANDOFF)` and asserts non-None, "05-02" present, and `\033[32m` (GREEN ANSI) present. The GREEN assertion is load-bearing: it distinguishes a live executing resolution from the DIM idle fallback (`\033[2m`) that the stale-timestamp bug produced.
