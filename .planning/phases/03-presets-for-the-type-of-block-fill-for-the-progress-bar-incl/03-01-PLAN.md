<objective>
Introduce the `bar_style` config key and the three whole-cell presets — `shade` (current default), `solid`, `solid-dim` — plus the new per-cell color rule (filled = threshold color, empty = dim gray). Refactor the hardcoded `_FILLED`/`_EMPTY` constants into a preset lookup, thread `bar_style` from config through `render_bottom_line` into `_context_segment`, and add silent fallback to `shade` for unknown values.

Purpose: Let the user opt into visually distinct context-bar fills via config (D-08/D-09), and sharpen all block-based presets with the filled/empty color split (D-06/D-07), while existing installs see zero change.
Output: Modified `claude-statusline.py`, updated tests in `test_nerd_icons.py` / `test_bottom_line.py`, and a `bar_style` config test in `test_config.py`.
</objective>
